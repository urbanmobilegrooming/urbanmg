## 1. COMPETITOR ANALYSIS

### 1.1 MoeGo (Primary Competitor)

**Overview:** The dominant all-in-one pet business operating system. Purpose-built for pet care (grooming, boarding, daycare). This is what urbanMG must beat.

**Pricing:**
| Plan | Cost | Key Differentiators |
|------|------|-------------------|
| Basic | $49/mo | 1 user, smart schedule, online booking, payments, reminders, digital agreements, mass text, no-show protection |
| Growth | $99/mo per van | Multi-van scheduling, route optimization, intelligent waitlist, advanced CRM, marketing, leads tracking, custom price model, multi-location sync |
| Ultimate | $159/mo per van | Unlimited automated messages, branded phone number, custom reporting, business insights, custom CSS/JS |
| Enterprise | Custom | Franchise tools, API platform, custom portal, enterprise BI, tailored onboarding |

All plans: unlimited clients, pets, appointments, devices.

**Key Features:**
- **MoeGo Smart Schedule:** Auto-suggests efficient time slots based on driving distance/time between appointments. When enabled, customers can only book slots that make route sense.
- **Route Optimization:** Uses map view to rearrange appointments by fastest driving time/shortest distance. User picks start/end points, system optimizes the middle. Manual drag-and-drop adjustment after optimization.
- **Map View:** Switch between map/calendar with one tap. Shows appointment locations on map. Real-time staff location tracking. Service area visualization per staff member. Driving time & distance displayed between appointments.
- **Online Booking (Mobile Grooming):** Address verification restricts bookings to service areas. Out-of-area clients can submit requests without times. Smart Scheduling limits availability by max driving distance/time. "Certain areas for certain days" settings.
- **Client Communication:** 2-way messaging center, mass texting, auto-reply, 3 types of automated reminders (reduces no-shows by 80%), ready-for-pickup notifications.
- **Payments:** Deposit collection, no-show fees, card on file, prepayments, next-day payouts, mobile POS, in-van checkout.
- **Staff Management:** Clock in/out, role permissions, commission & payroll tools.
- **Leads Management:** Abandoned booking recovery, leads tracking.
- **Pet Profiles:** Breed, coat details, grooming history, behavioral notes, medical alerts, owner preferences, vaccination tracking.
- **Digital Agreements:** Electronic consent signing.
- **Google Reserve Integration:** Direct booking from Google Search.

**MoeGo Weaknesses (urbanMG Opportunities):**
- SMS text limits: Growth plan only 900 SMS/month for ALL automated + 2-way messages. Exceeding = pay more.
- Reports are "super limited" per user reviews.
- Can't disable booking for specific groomers on certain dogs.
- Limited room for grooming notes and dog info.
- No text/email to clients unless you pay extra.
- App doesn't load well without full phone reception.
- Customizing pricing across breeds is cumbersome.
- Manual sorting of service/staff lists (no auto-sort).
- Steep learning curve for new users.
- Onboarding process is rushed (single data import, no staged go-live).
- Limited payment-splitting capability.
- Time clock error correction is difficult.
- Customer support response can be slow.
- No WhatsApp integration (SMS only, big deal for Miami's Latin market).
- No multi-language support (EN/ES/PT).

## 4. TECHNOLOGY INTEGRATION RESEARCH

### 4.1 Google Calendar API

**How it works for appointment booking:**
- Create Google Cloud Project, enable Calendar API
- Use service account for server-to-server auth
- Real-time availability sync prevents double-bookings
- Webhooks for change notifications (no polling needed)
- Business hours enforcement, holiday awareness
- Timezone handling

**Best Practices:**
- Use webhooks (not polling) for real-time sync
- Service account with "See all events details" permission
- Email verification to prevent spam bookings
- Handle rate limits gracefully
- Normalize data across calendar sources

**Implementation:** Use `googleapis` npm package. Create events via `calendar.events.insert()`. Watch for changes via `calendar.events.watch()`.

---

### 4.2 Stripe Payment Integration

**For Deposits & Card on File:**
- **Setup Intent:** Save card for future use without charging (card on file)
- **Payment Intent with capture_method: 'manual':** Pre-authorize (hold) funds, capture later
  - Online card holds valid for 7 days
  - In-person Terminal holds valid for 2 days
  - `capture_before` attribute indicates expiration
- **Supported methods for holds:** Cards, Affirm, Afterpay, Cash App Pay, Klarna, PayPal

**For Recurring Payments/Subscriptions:**
- Use Stripe Billing for membership packages
- `stripe.subscriptions.create()` for recurring
- Webhook: `checkout.session.completed` for order fulfillment
- Webhook: `invoice.payment_succeeded` for subscription renewals

**For One-Time Payments:**
- Stripe Checkout (hosted page) or Payment Element (embedded)
- Apple Pay / Google Pay support built-in
- Next-day deposits standard, instant for additional fee

**Best Practices with Next.js:**
- Server-side: Create PaymentIntents in API routes
- Client-side: Use `@stripe/react-stripe-js` for UI
- Always verify payments via webhooks (not redirect callbacks)
- PCI compliance handled by Stripe Elements

### 4.5 Route Optimization APIs

#### Option A: Google Routes API
- **Compute Routes:** Basic $5/1000 requests, Advanced $10/1000
- **Route Optimization API:** Per-shipment pricing, Single Vehicle or Fleet Routing SKUs
- Supports time windows, vehicle capacity, driver hours constraints
- Max 98 waypoints per request (lat/lng) or 25 (place IDs)
- Real-time traffic data

#### Option B: Google Route Optimization API
- Dedicated fleet routing
- Define business objectives (travel efficiency, on-time arrival)
- Define constraints (driver hours, vehicle capacity)
- Single-vehicle and multi-vehicle routing
- Accepts shipment details, time windows, capacity

#### Option C: OSRM (Self-Hosted, Open Source)
- **Cost:** ~$50/month server vs pay-per-request Google
- **Performance:** Sub-millisecond queries for continental-scale data
- **Services:** Route, Table (distance matrix), Trip (TSP solving), Match
- **Data:** OpenStreetMap (free, often more current than proprietary)
- **Customization:** Custom routing profiles (avoid tolls, prefer highways)
- **Trade-off:** Heavy preprocessing; map updates take time
- **Best for:** High-volume routing, cost control, data ownership

#### Recommendation for urbanMG:
Start with **Google Routes API** (simple, reliable, real-time traffic). For 2 vans with ~8 stops each:
- ~2 optimization requests/day = ~60/month
- Cost: essentially free within $200/month Google Maps credit
- Switch to OSRM later if scaling to many vans

### 4.6 Recurring Appointments (RRULE)

**Standard:** iCalendar RFC 5545 RRULE format

**Key Parameters:**
- `FREQ`: DAILY, WEEKLY, MONTHLY, YEARLY
- `INTERVAL`: Units between occurrences (e.g., INTERVAL=4 with FREQ=WEEKLY = every 4 weeks)
- `BYDAY`: MO, TU, WE, TH, FR, SA, SU
- `COUNT`: Total number of occurrences
- `UNTIL`: End date
- `BYSETPOS`: Position within set (e.g., 1st Monday)

**Example for Pet Grooming:**
- Every 4 weeks on same day: `FREQ=WEEKLY;INTERVAL=4;BYDAY=TH`
- Every 6 weeks: `FREQ=WEEKLY;INTERVAL=6;BYDAY=MO`

**Implementation:**
- Use `rrule` npm package to parse/generate
- Store RRULE string in database alongside appointment
- Generate future instances on-demand
- Allow individual instance overrides (reschedule one without affecting series)
- Auto-send rebooking reminders when no next appointment exists

**Grooming-Specific Patterns:**
- Full groom: every 4-6 weeks
- Bath: every 2-4 weeks
- Puppy first groom: one-time, then switch to recurring
- Seasonal (summer shave-downs): annual pattern

### 4.7 Online Intake Forms & Digital Agreements

**What to Collect:**
1. **Client Intake Form:** Name, address, phone, email, preferred language, preferred contact method, how they found us
2. **Pet Profile Form:** Name, breed, age/DOB, weight, coat type, temperament, medical conditions, allergies, medications, vaccination records (upload), vet info, behavioral notes
3. **Service Agreement/Waiver:** Liability waiver, matting policy, health condition disclosure, flea/tick policy, payment terms, cancellation policy, photo release consent, e-signature capture

**Implementation Options:**
- **Custom-built** (recommended): Next.js forms with Supabase storage. Full control, no third-party dependency, data stays in your system.
- **E-signature:** Use `signature_pad` npm library for touch-friendly signatures on mobile
- **PDF generation:** Use `@react-pdf/renderer` for generating signed agreement PDFs
- **Storage:** Supabase Storage for uploaded files (vaccination records, photos)
- **Auto-send:** Trigger via WhatsApp or email before first appointment
